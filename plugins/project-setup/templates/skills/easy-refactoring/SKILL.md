# 리팩토링 규칙

## 리팩토링 허용 범위

- 변수/함수 이름 개선
- 중복 코드 추출 (DRY)
- 복잡한 조건문 → 서브 함수 분리
- 불필요한 코드/주석 제거

## 리팩토링 금지 조건

- 외부 동작(API 응답 형식) 변경 → 금지
- 컴포넌트 props 인터페이스 변경 → 금지
- DB 스키마 변경 → 금지
- 리팩토링과 기능 추가를 동시에 → 금지 (커밋 분리)

## 리팩토링 트리거

- 같은 로직 3회 이상 반복 → 공통 함수 추출
- 함수 50줄 초과 → 분리 검토
- 파라미터 4개 이상 → 객체로 묶기
- 중첩 if 3단계 이상 → early return 패턴

## 리팩토링 후 필수

- 기존 테스트 전부 통과 확인
- 타입 체크 통과 확인
- 린트 통과 확인
